Concept 2 / The Lantern

A quieter editor where advanced behaviour remains mostly hidden until the user needs it, then appears beside the selected node as small contextual lantern cards.

The Lantern aims for lower stress. Remote logic and semantic actions are still explicit, but they reveal themselves locally through anchored mini-cards around the selection. It feels less like switching tools and more like the canvas illuminating its own meaning.

Why it works

Behaviour without mode debt

Contextual
Action peeks
Selection reveals the semantic action right beside the node, so stream selection feels understandable without a form-heavy binding table.
Route peeks
Only the active node’s relevant neighbours appear, which keeps the canvas calm unless the user is actively testing navigation.
Compile peeks
In-place hints warn when the user is drifting toward awkward density or unsupported decoration.
Good fit
Strong for setup menus, audio and subtitle selectors, and lower-density pages where local clarity matters more than seeing the whole route graph at once.
