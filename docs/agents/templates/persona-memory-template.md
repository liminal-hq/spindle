# Persona Memory Template

## Purpose

This file is a working memory note for a Spindle persona.

Use it to capture durable context that should survive across multiple conversations, implementation passes, and review cycles.

The goal is structural memory, not narration.

## How To Use This File

Update this file when you learn something likely to matter later, especially:

- repository facts that change how work should begin
- stable architecture boundaries
- durable product or optical constraints
- Rust, Tauri, React, or TypeScript contract realities
- recurring review traps
- verification expectations that future passes should repeat
- handoff patterns that reliably reduce re-orientation cost

Do not turn this into a diary.

Prefer short bullets containing durable facts over long session narratives.

## Current Notes

- Add stable repository and product facts here.
- Prefer concrete paths, contracts, and constraints over speculation.
- Record only the details that a future pass would genuinely need.

## Open Questions

- Add unresolved questions here.
- Keep them narrow enough that a future pass can act on them.
- Remove or convert them into durable notes once they are resolved.
